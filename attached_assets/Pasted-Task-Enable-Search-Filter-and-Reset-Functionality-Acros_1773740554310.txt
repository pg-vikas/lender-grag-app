Task: Enable Search, Filter, and Reset Functionality Across Pages with Special Handling for Growth KP Page

Requirements:

1. The **Search, Filter, and Reset** functionality should work on the following pages:
   - Clients Page
   - Growth Page
   - Launchpads Page
   - Invoices Page
   - User Page
   - Growth KP Page

2. **Search Behavior**
   - When a user types text in the **Search input**, the value should be captured properly.
   - Clicking the **Filter** button should display data filtered according to the search input.

3. **Page-Specific Filtering**
   - Each page should apply **its own filtering logic** based on the data displayed on that page.
   - Filtering should only affect the **data shown on the current page** and should not impact other pages.

4. **Reset Button**
   - Clicking the **Reset** button should:
     - Clear the search input field.
     - Reset any applied filters.
     - Restore the default data list for that specific page.

5. **Growth KP Page Special Condition**
   - The **Growth KP page** has a **different filter design** compared to other pages.
   - **Do not modify the existing design, layout, or styling** of the filter on the Growth KP page.
   - Only implement/fix the **search filter functionality** so that it works correctly with the current design.

6. **Restrictions**
   - Do **not change the UI design** or layout of any existing filter components.
   - Do **not affect any other functionality** on the pages.
   - Ensure the search and filter logic works while keeping the **current designs unchanged**.

7. **UI Consistency**
   - Maintain the existing dashboard styling, spacing, hover effects, and responsiveness.
   - Ensure all search inputs, filter buttons, and reset buttons remain clearly visible and functional.

Expected Result:

- Search and Filter work correctly on **Clients, Growth, Launchpads, Invoices, User, and Growth KP pages**.
- Each page filters data **based only on its own dataset**.
- Clicking **Reset** clears the search input and restores the default data list.
- The **Growth KP page filter design remains unchanged**, but the search functionality works properly.
- No other UI elements or functionality are affected.
- All controls work smoothly and consistently with the dashboard UI.
