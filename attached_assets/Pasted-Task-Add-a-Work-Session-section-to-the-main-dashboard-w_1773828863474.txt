Task: Add a **Work Session section** to the main dashboard with Clock In / Clock Out timer functionality.

Requirements:

1. **Work Session Section UI**
   * Add a new section on dashboard: **“Work Session”**.
   * Include:
     - Clock In button
     - Clock Out button
     - Live Timer display (HH:MM:SS)
     - Clock In Time display
     - Clock Out Time display
     - Break Time display

2. **Clock In Functionality**
   * Clicking **Clock In**:
     - Starts the timer
     - Stores current time as **Clock In Time**
     - Starts live elapsed time counter
   * If already clocked in, button should be disabled.

3. **Clock Out Functionality**
   * Clicking **Clock Out**:
     - Stops the timer
     - Stores current time as **Clock Out Time**
     - Stops live counter
   * If not clocked in, button should be disabled.

4. **Timer Behavior**
   * Timer runs continuously while user is clocked in.
   * Display **real-time elapsed duration** since last Clock In.
   * Format: HH:MM:SS.

5. **Break Time Calculation**
   * If user:
     - Clocks Out → then Clocks In again
     - The gap between these times is counted as **Break Time**
   * Accumulate total break duration across the day.

6. **Multiple Sessions in a Day**
   * Support multiple Clock In / Clock Out cycles.
   * Track:
     - Total working time (sum of all active sessions)
     - Total break time (sum of all inactive gaps)

7. **Daily Reset**
   * Timer resets automatically at **12:00 AM (new day)**:
     - Clear all session data (Clock In, Clock Out, Break, Timer)
   * Ensure new day starts fresh.

8. **Live Display**
   * While clocked in:
     - Timer updates every second
     - Shows **current session elapsed time**
   * While clocked out:
     - Timer stops

9. **State Management**
   * Store:
     - Current session start time
     - Previous sessions
     - Break durations
   * Data should persist (optional: localStorage for refresh safety).

10. **UI & Theme**
   * Match **dashboard styling** (cards, spacing, typography).
   * Buttons should have active/disabled states.
   * Timer should be clearly visible and prominent.

Expected Result:

* Clicking **Clock In** starts timer and shows live elapsed time.
* Clicking **Clock Out** stops timer and records time.
* Break time is calculated automatically between sessions.
* Multiple sessions per day are handled correctly.
* Timer resets automatically at midnight.
* UI is clean, responsive, and consistent with dashboard.
