Build a production quality admin module called Pre Approval Letters for Lender Greg’s backend admin dashboard inside Gorilla Apps.

This module is used by a mortgage lender to manage pre approved borrowers and generate polished branded pre approval letters quickly.

The module must feel like a serious internal financial tool. The design should be modern, clean, sharp, premium, and efficient. It should look like software used by a real lender, not a consumer app. Prioritize clarity, confidence, speed, and professional structure.

Primary Goal

Create a module where Greg can:

view a list of pre approved borrowers
search and filter them
select a borrower
input or update a few deal specific fields
instantly preview a branded pre approval letter
generate a PDF
email the letter
save the version to borrower history

This module should reduce the workflow to a few clicks.

Functional Behavior

Implement the module with realistic internal admin behavior.

Required behavior
selecting a borrower loads all borrower data into the form
updating form fields changes live preview instantly
generate PDF button creates a downloadable formatted PDF
save version button creates a new history record
send button opens modal for email delivery
archived borrowers should not be editable unless restored
Validation rules
purchase price cannot exceed max approved purchase price unless override enabled
loan amount cannot exceed max loan amount unless override enabled
expiration cannot be earlier than issue date
required borrower name and loan program must exist before generating
if approval expired, show blocking warning before final generation
